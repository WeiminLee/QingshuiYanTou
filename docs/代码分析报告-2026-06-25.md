### 1. StockScore 悬空列 — `backend/app/models/models.py:476-488`

`StockScore` 类中包含 9 个不属于该模型的列定义，注释写着"已迁移至 Neo4j"但列被遗留在 `StockScore` 中，数据库中也不存在这些列：

```python
# 这些列位于 StockScore 类内部（models.py:476-488）
id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
sync_type: Mapped[str] = mapped_column(String(20), nullable=False)
last_sync_at: Mapped[datetime | None] = mapped_column(DateTime)
records_synced: Mapped[int] = mapped_column(Integer, default=0)
announcement_count: Mapped[int] = mapped_column(Integer, default=0)
report_count: Mapped[int] = mapped_column(Integer, default=0)
status: Mapped[str] = mapped_column(String(20), default="success")
info: Mapped[str | None] = mapped_column(Text)
synced_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
```

- **影响**: 重复 `id` 主键导致 SQLAlchemy ORM 初始化失败，`stock_scores` 表的任何查询都报错
- **根因**: 原 `kg_entities`/`kg_relationships` 表（已删除）的列被误留在 `StockScore` 类中
- **修复**: 删除 L476-488 的 9 行

### 12. `neo4j_client.py` 中 TYPE_CHECKING 导入 + 模块级类型注解 — `backend/app/core/neo4j_client.py:24,63`

```python
from __future__ import annotations  # L8
_async_driver: AsyncDriver | None = None  # L24 — AsyncDriver 未定义
...
if TYPE_CHECKING:                        # L63
    from neo4j import AsyncDriver         # L64
```

- **影响**: 虽然 `from __future__ import annotations` 让注解变成惰性字符串，不会运行时崩溃，但代码很脆弱：
  - 如果移除 `from __future__ import annotations`，L24 立即抛 `NameError`
  - 对维护者造成困惑（`AsyncDriver` 看起来是未定义变量）
- **修复**: 将 `from neo4j import AsyncDriver` 移到 TYPE_CHECKING 块外，或将注解改为字符串 `"AsyncDriver | None"`
